# Phase 2 设计方案：字源体验优化 + HSK 词库扩充 + 会员收费系统

## 一、字源演变体验优化

### 现状问题

当前字源区只有：一个古字符号 → 箭头 → 现代汉字 + 一段文字说明。
过于简单，用户看不到"从古到今一步步演变"的过程，记忆效果有限。

### 优化方案：分阶段演变时间轴

将每个汉字的演变拆成 **4-5 个阶段**，以横向时间轴的方式展示：

```
甲骨文 → 金文 → 篆书 → 楷书（现代）
 ☉      ⊙      日      日
(约前1300年) (约前800年) (约前200年)  (现在)
```

#### 交互设计

1. **时间轴卡片**：每个阶段是一张小卡片，包含：
   - 该阶段的字形图片（从开源数据集获取）
   - 阶段名称（甲骨文/金文/篆书/楷书）
   - 大致年代
   
2. **逐步动画揭示**：不一次性展示所有阶段，而是让用户点击"下一步"逐步查看演变过程，每一步之间有一个平滑的变形过渡动画

3. **故事化说明**：每个阶段配一句简短的泰语说明，解释这个阶段的字形为什么长这样。例如：
   - 日（甲骨文）："คนโบราณวาดรูปวงกลมมีจุดตรงกลาง เพราะมองดวงอาทิตย์แล้วเห็นเป็นวงกลมสว่าง"
   - 日（金文）："ต่อมาวงกลมเริ่มเปลี่ยนเป็นสี่เหลี่ยม เพราะเขียนบนสำริดได้ง่ายกว่า"
   - 日（楷书）："สุดท้ายกลายเป็นรูปสี่เหลี่ยมมีเส้นขวาง คือตัว 日 ที่เราใช้ในปัจจุบัน"

#### 图片资源方案

30 个基础汉字 × 3 个历史阶段 = 约 90 张图片。两种方案：

- **方案 A（推荐）：SVG 手绘风格**
  自己绘制简化的古文字 SVG，风格统一，加载快，不依赖外部版权。
  工具：Figma / Inkscape 批量制作，或用代码生成简笔 SVG path。

- **方案 B：开源数据集裁剪**
  从 character-Evolution-Dataset 中提取，需要裁剪和统一尺寸。
  优点：真实感强。缺点：图片风格不统一，版权不明确。

## 二、HSK 词库扩充计划

### HSK 词汇量概览

| 等级 | 词汇量（约） | 新增汉字（约） | 定位 |
|------|-------------|---------------|------|
| 免费体验 | 30 字 | 30 | 入门体验，吸引用户 |
| HSK 1 | 150 词 / 174 字 | ~150 | 基础生存交流 |
| HSK 2 | 150 词 / 173 字 | ~150 | 简单日常会话 |
| HSK 3 | 300 词 / 270 字 | ~270 | 基本生活沟通 |
| HSK 4 | 600 词 / 430 字 | ~430 | 较流利日常交流 |

### 内容组织方式

每个 HSK 等级内部，按主题分组（而非死板地按词汇表顺序）：

```
HSK 1 示例分组：
├── 数字与时间 (Day 1-3)
├── 家庭与人称 (Day 4-5)
├── 日常物品 (Day 6-8)
├── 基础动词 (Day 9-12)
├── 地点与方位 (Day 13-15)
└── ...
```

每组 5-8 个字，每天学一组，保持节奏感。

### 数据结构扩展

```json
{
  "id": 31,
  "char": "好",
  "pinyin": "hǎo",
  "thai": "ดี",
  "level": 11,
  "group": 11,
  "groupName": "คำคุณศัพท์พื้นฐาน",
  "hsk": 1,
  "isFree": false,
  "audio": "audio/好.mp3",
  "etymology": { "stages": [...] }
}
```

新增字段：
- `hsk`: 所属 HSK 等级（0 = 免费体验包）
- `isFree`: 是否免费可用

## 三、会员收费系统设计

### 用户分层

| 身份 | 可学内容 | 功能权限 |
|------|---------|---------|
| **游客（未注册）** | 30 个免费汉字 | 全部学习功能，进度存本地 |
| **免费会员（注册）** | 30 个免费汉字 | 进度云端同步，多设备使用 |
| **付费会员** | 解锁 HSK 1-4 全部内容 | 云端同步 + 全部词库 |

### 收费模式选项

#### 方案 A：按 HSK 等级单独购买
- HSK 1 解锁：¥29 / 199 ฿（一次性）
- HSK 2 解锁：¥39 / 269 ฿
- HSK 3 解锁：¥49 / 339 ฿
- HSK 4 解锁：¥59 / 409 ฿
- 全套打包：¥129 / 899 ฿（优惠约 27%）

**优点**：用户可按需购买，降低付费门槛
**缺点**：实现复杂，需要管理多个购买状态

#### 方案 B：订阅制（月/年）
- 月会员：¥19 / 129 ฿
- 年会员：¥149 / 999 ฿（相当于月付 ¥12.4）

**优点**：持续收入，实现简单
**缺点**：用户可能抗拒订阅

#### 方案 C（推荐）：混合制
- 单等级购买（永久）：各等级单独定价
- 年度会员（全部解锁）：年费享全部内容
- 好处：给用户选择权，轻度用户买单等级，重度用户订阅更划算

### 支付方式

目标用户是泰国学生，需要支持泰国主流支付：
- **PromptPay**（泰国最普及的即时支付）
- **TrueMoney Wallet**
- **Credit/Debit Card**
- 支付网关推荐：**Omise**（泰国本土支付网关）或 **Stripe**（支持泰国）

### 技术实现方案

当前是纯前端静态站，加入收费系统需要后端。两种路径：

#### 路径 1：轻量后端（推荐起步方案）
```
前端（现有）  ←→  Supabase / Firebase
                    ├── 用户认证（邮箱/Google/Line登录）
                    ├── 用户数据（学习进度云端同步）
                    ├── 会员状态（哪些 HSK 已解锁）
                    └── 支付回调验证
```
- **Supabase**：免费额度大，PostgreSQL，自带 Auth
- **Firebase**：Google 生态，免费额度也够用
- 前端仍然部署在 GitHub Pages，后端用 BaaS 服务

#### 路径 2：全栈方案（长期）
```
React/Vue 前端  ←→  Node.js API  ←→  PostgreSQL
                        ├── 用户系统
                        ├── 支付集成
                        ├── 内容管理
                        └── 学习数据分析
```
适合用户量大了以后迁移。

### 内容锁定机制（前端）

```javascript
  const charData = characterData.find(c => c.level === level);

    showPaywall(charData.hsk);  // 弹出解锁提示
    return;
  }
  
  // 正常进入学习...
}

  // 显示：
  // - 该 HSK 等级包含多少个字
  // - 价格
  // - 购买/订阅按钮
  // - 免费试学 3 个字的入口（让用户尝鲜）
}
```

### Dashboard UI 变化

被锁定的 HSK 等级卡片：
- 半透明 + 锁头图标
- 点击后弹出解锁/购买提示
- 显示"试学 3 个字"入口（降低决策压力）

## 五、关键决策点（需要讨论）

1. **收费模式**：按等级买断 vs 订阅制 vs 混合？
2. **支付渠道**：Stripe（国际化）vs Omise（泰国本土）vs 两者都接？
3. **后端选择**：Supabase vs Firebase？
4. **用户登录方式**：邮箱、Google、Line（泰国主流社交）？
5. **字源图片**：自己画 SVG vs 从数据集提取？
6. **免费试学**：锁定内容是否提供"试学 3 个字"的体验？
